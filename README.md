# dcf77emu

Cross-platform Python emulator of the **DCF77** longwave time signal (77.5 kHz)
using nothing but a standard PC audio output. Lets you synchronize a
radio-controlled clock (Citizen, Junghans, Braun, Casio Wave Ceptor, …)
without a real DCF77 transmitter or hardware modulator.

Runs on macOS, Linux and Windows. Managed entirely with [uv](https://docs.astral.sh/uv/).

## How it works

DCF77 broadcasts the official German atomic time on a **77.5 kHz** longwave
carrier. That frequency is far above any consumer sound card's Nyquist
limit (24 kHz at 48 kHz sample rate), so it cannot be played as audio.

Two mechanisms can put energy on 77.5 kHz from an audio jack:

**1. Image mode (default).** DAC zero-order-hold imaging. When a DAC
outputs a sampled tone at frequency `f`, it produces spectral mirror
images at `k·fs ± f` for every integer `k`. At 48 kHz:

```
2·fs − f = 77500   ⇒   f = 96000 − 77500 = 18500 Hz
```

So the emulator emits **18.5 kHz** out of the jack and the DAC's image at
`2·48000 − 18500 = 77500 Hz` is what the clock receives.

**2. Harmonic mode (`--mode harmonic`).** Emit `77500/k` Hz so the
**k-th harmonic** generated by speaker/cable nonlinearity lands on
77.5 kHz. At 48 kHz the smallest viable fundamental is `77500/4 = 19375 Hz`
(k=4). Lower `k` → less harmonic distortion required, higher pitch.
`k=5` → 15.5 kHz (DCF77.js-style, more audible but rich harmonics).

DCF77 amplitude modulation is reproduced on top of this carrier by
attenuating it to ~15 % during the first 100 ms (bit 0) or 200 ms
(bit 1) of every second. Second 59 carries no pulse so the clock can
align on the minute boundary.

NTP synchronization runs in a background thread so the transmitted time
stays accurate even if the local system clock drifts.

## Install

### Option A — uv (recommended)

Install [uv](https://docs.astral.sh/uv/) once:

```sh
# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh
# Windows (PowerShell)
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"
```

Clone and sync:

```sh
git clone https://github.com/Giorgiofox/dcf77emu.git
cd dcf77emu
uv sync
```

### Option B — plain pip + venv

If you don't want `uv`:

```sh
git clone https://github.com/Giorgiofox/dcf77emu.git
cd dcf77emu
python3 -m venv .venv
source .venv/bin/activate         # Windows: .venv\Scripts\activate
pip install numpy sounddevice tzlocal ntplib
```

Then invoke `python3 main.py ...` everywhere this README shows `uv run dcf77emu ...`.

### Linux system dependency

`sounddevice` wraps PortAudio. On Debian/Ubuntu:

```sh
sudo apt update
sudo apt install -y libportaudio2
# if still missing on some distros:
sudo apt install -y libportaudio2 libportaudiocpp0 portaudio19-dev
```

Verify:

```sh
ldconfig -p | grep portaudio       # must show libportaudio.so.2
```

On Fedora/RHEL: `sudo dnf install portaudio`. On Arch: `sudo pacman -S portaudio`.
macOS and Windows wheels bundle PortAudio — no system install needed.

## Run

### Quick start

```sh
uv run dcf77emu                          # default: image mode, square wave, NTP on
uv run dcf77emu --list-devices           # list audio outputs first
uv run dcf77emu -d 3                     # pick output device by index
```

### Timezone & NTP

```sh
uv run dcf77emu -t Europe/Rome           # override timezone
uv run dcf77emu --no-ntp                 # disable NTP, use system clock only
uv run dcf77emu --ntp-server time.nist.gov
uv run dcf77emu --ntp-refresh 600        # resync every 10 min
```

### Wave shape

```sh
uv run dcf77emu -w sine                  # cleaner spectrum, weaker field
uv run dcf77emu -w square                # default, strongest field
uv run dcf77emu -w pulse                 # narrow-pulse train, loudest audible
uv run dcf77emu -w quant                 # quantized sine, harmonic-rich
```

### Carrier mode

```sh
uv run dcf77emu -m image                 # default DAC-imaging at 18.5 kHz
uv run dcf77emu -m harmonic              # 4th harmonic (19.375 kHz fund.)
uv run dcf77emu -m harmonic --harmonic-k 5   # 5th harmonic (15.5 kHz, DCF77.js-style)
uv run dcf77emu -m harmonic --harmonic-k 7   # 7th harmonic (~11.07 kHz, more audible)
```

### Audibility / spectrum tweaks

```sh
uv run dcf77emu --lowshelf               # cut audible band <12 kHz (quieter whistle)
uv run dcf77emu -m harmonic -w quant --lowshelf   # gentlest combo
uv run dcf77emu -m harmonic -w pulse              # most aggressive harmonics
```

### Manual overrides

```sh
uv run dcf77emu -c 18500                 # force custom audio carrier (Hz)
uv run dcf77emu -r 96000                 # force sample rate
uv run dcf77emu -r 96000 -c 77500        # if your DAC supports it, emit 77.5 kHz directly
```

### Plain-Python equivalents (no uv)

```sh
source .venv/bin/activate
python3 main.py --list-devices
python3 main.py -m harmonic --harmonic-k 5
python3 main.py -t Europe/Rome --no-ntp
```

Live status line during operation:

```

audio carrier: 18500.00 Hz  (image (k=2, sign=-))
wave shape:    square
target RF:     77500 Hz  -> DCF77
timezone:      Europe/Rome
ctrl-c to stop
2026-05-14 16:23:07 Europe/Rome  sec=07 bit=1 pulse=200ms  NTP=  +12.3 ms
```

## Practical setup

1. Set **system volume to maximum**.
2. Plug in **headphones or a 3.5 mm cable** — the cable is the antenna.
   Built-in laptop speakers radiate far less.
3. Lay the clock on the cable or within ~5 cm of the jack.
4. Put the clock in **manual reception mode** (consult its manual). It
   typically takes 2–5 minutes to lock on; some clocks need a full
   minute frame plus parity verification.
5. If your clock shows a signal-strength indicator (L / M / H), aim for
   M or H by repositioning the cable and clock.

## Wave shapes

| Shape   | Spectrum                        | Field at 77.5 kHz | Audible noise           |
| ------- | ------------------------------- | ----------------- | ----------------------- |
| sine    | single tone                     | weakest           | quietest                |
| square  | odd harmonics                   | strong (default)  | high whistle            |
| pulse   | dense harmonic content          | strongest         | loudest, harsher        |
| quant   | sine quantized to 8 levels      | medium-strong     | between sine and square |

If your clock won't lock with the default `square`, try `pulse`. If the
audible tone bothers you and the field is already enough, drop to `sine`
or pass `--lowshelf` to attenuate the audible band while keeping the
high carrier.

## Carrier modes

| Mode (`-m`) | Mechanism                              | Pros                          | Cons                          |
| ----------- | -------------------------------------- | ----------------------------- | ----------------------------- |
| `image`     | DAC zero-order-hold image at 77.5 kHz  | exact frequency, predictable  | depends on DAC reconstruction |
| `harmonic`  | k-th harmonic from speaker nonlinearity| works on noisy/cheap audio    | needs distortion, weaker      |

Use `harmonic` mode if the default `image` mode fails — different audio
hardware leaks differently. Combine with `square`/`pulse` for richer
harmonic content.

## CLI reference

```
--timezone, -t       IANA timezone (default: system timezone)
--carrier,  -c       Force audio carrier in Hz (default: auto)
--mode,     -m       Carrier mode: image | harmonic (default: image)
--harmonic-k         Force harmonic order k for --mode harmonic
--wave,     -w       Wave shape: sine | square | pulse | quant (default: square)
--lowshelf           Attenuate audible band <12 kHz via biquad
--device,   -d       Output device index (see --list-devices)
--sample-rate, -r    Force sample rate (default: 48000)
--list-devices       List audio devices and exit
--ntp-server         NTP server (default: pool.ntp.org)
--ntp-refresh        NTP refresh interval in seconds (default: 3600)
--no-ntp             Disable NTP sync
```

## Troubleshooting

**`uv: command not found`** — install uv first (see Install above) or use the plain pip route.

**`OSError: PortAudio library not found`** — install `libportaudio2` (see Linux system dependency).

**`--list-devices` shows nothing** — headless server / no sound card. The emulator needs a physical audio jack to radiate the signal. Run it on a laptop/desktop, not a remote server.

**Clock won't lock**
- Increase system volume to max.
- Plug in headphones/3.5 mm cable — built-in speakers radiate far less.
- Try `-m harmonic` if `image` (default) fails: different hardware leaks differently.
- Try `-w pulse` or `-w square` for more harmonic energy.
- Verify time/timezone are correct via the live status line.

**Audible whistle too loud** — pass `--lowshelf`, or switch to `-w sine`, or use harmonic mode with higher `k` (which lowers the fundamental into more audible territory but lets you turn volume down).

## Files

- `dcf77.py` — protocol encoder (BCD, parity, minute frame layout)
- `signal_gen.py` — sine / square / pulse / quant generators, lowshelf biquad, DCF77 envelope
- `timesync.py` — background NTP client
- `main.py` — CLI, carrier picker (image + harmonic), audio stream, second-by-second scheduling

## Disclaimer

This emulator radiates an extremely low-power field measured in nanowatts,
intended for synchronizing clocks at desk distance. It is not a
transmitter and is not intended for broadcasting. Local radio regulations
may apply; you are responsible for compliant use.
